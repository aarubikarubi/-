# 🗺️ DailyGameLauncher 中長期改善ロードマップ

---

## フェーズ 1：技術的堅牢性とドキュメントの整備 ✅

> **目標:** 安定した土台を作り、プロジェクトを長期にわたってメンテナンスしやすい状態にする。

### 1-A. リポジトリの信頼性と認知度の向上

| 項目 | 内容 |
|------|------|
| **ドキュメントの品質向上** | READMEやリリースノートを自然で分かりやすい文章に整備する。 |
| **メタデータの充実** | GitHubの「About」セクションに適切な説明文とトピックタグ（`python`, `game-launcher`, `pyqt6` など）を設定し、検索性を高める。 |
| **自動ビルドの導入** | GitHub Actionsを活用して、プッシュ時に自動でインストーラー（.exe）を生成・リリースするパイプラインを構築する。 |

### 1-B. 技術的な堅牢性とメンテナンス性の強化

| 項目 | 内容 |
|------|------|
| **ディレクトリ構造の整理** | ソースコードを `src/`、リソースを `assets/`、テストを `tests/` に分離して整理する。 |
| **ロギングの導入** | `core.py` に、ゲームの起動・終了やエラーの詳細を記録するログ出力機能を追加する。 |
| **依存関係の明示** | `requirements.txt` を作成して必要なライブラリを明記する。 |

## フェーズ 2：UI/UX（ユーザー体験）のブラッシュアップ 🎨

> **目標:** ユーザーが迷わず、直感的に操作できるインターフェースを実現する。

### 2-A. ツールチップ（簡易マニュアル）の実装

PyQt6の `QToolTip` を活用して、各設定項目に以下の説明文を表示する：

| 対象 UI | ツールチップ内容 |
|---------|----------------|
| 「次のゲームを開くまでの待ち時間」 | "PCのカクつきを防ぐための準備時間です。性能が低いPCの場合は長めに設定してください。" |
| 「一緒に閉じる裏方アプリ」 | "ゲーム終了時に、SteamやHoYoPlayなどの重いランチャーを自動で終了させてPCを軽くします。" |
| 「日課を開始」ボタン | "登録したリストの順番にゲームを自動起動します。" |
| 「要望・バグ報告」ボタン | "ブラウザを開いて開発者にメッセージやエラーログを送信します。" |

### 2-B. アイコン自動取得

登録した `.exe` ファイルからゲームアイコンを自動で抽出し、ゲームリストに表示する。

### 2-C. ドラッグ＆ドロップ対応

`.exe` ファイルをウィンドウにドロップするだけでゲームを登録できるようにする。

## フェーズ 3：機能拡張（高度な自動化） ✅

> **目標:** パワーユーザーのニーズに応える、高度な自動化機能を提供する。

### 3-A. ゲームプロファイル機能

「朝の日課用」「週末用」など、複数のゲームリストを保存・切り替え可能にする。

### 3-B. 負荷検知起動（スマート待機）

単なる秒数指定ではなく、**CPU負荷が一定値以下に下がったタイミング**で次のゲームを自動起動する「スマート待機」を実装する。

### 3-C. 常駐・グローバル操作の強化

- **タスクトレイ常駐:** ウィンドウを閉じてもバックグラウンドで動作を継続する。
- **グローバルホットキー:** ゲームを跨いで「スキップ」などの操作をキーボードショートカットで実行できる。
